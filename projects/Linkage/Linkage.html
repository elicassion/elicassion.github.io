Linkage

Interactive Search and Visualization of Knowledge Graph

Abstract

Knowledge graph, as a newly proposed framework for organizing data information in recent years, elaborates its natural structure in representing relationship between entities and relationships to describe data. Knowledge bases constructed using knowledge graph can be queried using a query language called SPARQL. However, the use of knowledge graph and SPARQL is difficult. In order to make querying procedure user-friendly, based on interactive data visualization, this paper designs and implements a system that can visualize the schema of a knowledge graph, perform input queries interactively by linking edges between nodes, and visualize the query results finally. The system is built to assist the input of most of the simple SPARQL queries.

Interaction Method

As stored in RDF format, I first get the Scheme, which is the definition of the structure of stored knowledge graph. It includes the Entities and Relations defined by the constructor. Therefore, I visualized the scheme by a graph, where Entities are represented by nodes and relations represented by edges are left blank for users to generate queries.

When the graph is ready, users can add edges between nodes, to identify a relation intended to query. These edges will be automatically converted to SPARQL codes describing relations. Corresponding to nodes involved in these relations, users are required to enter the content they want to specify by text. Involved nodes without entering texts are regarded the content the user wants to inquire. After input, the graph and associated texts can be converted to SPARQL query language, which will further be sent to the knowledge graph database to get the result.

For example, a simple knowledge graph has two Entities People and Object, descriping the belongings of each project. They are visualized as two nodes. The user wants to know which person possesses an apple. In the Linkage system, the user can drag to establish an edge starting from People node to the Object node. In the input fields below, the user enters "apple" corresponding to Object node. Then, SPARQL language snippte are generated and sent to the database to perform a query back. Using the similar nodes and edges method, tuples in the result can be visualized.

System Implementation

All of components, including interaction, visualization and converting to SPARQL language are written in JavaScript, using D3.js framework.
